Configuration source map

Use this as the operating rule for where settings should live. Secrets and infrastructure stay in environment variables; business rules that change during normal platform management live here.

Configuration area Source of truth Examples Reason
Secrets and credentials Environment only OPENAI_API_KEY, STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET, AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, SHARE_COOKIE_SECRET These must not be exposed in browser-editable platform screens.
Deployment and infrastructure Environment only BASE_URL, AWS_REGION, S3_BUCKET, S3_PREFIX, CORS_ALLOW_ORIGINS, APP_ENV These are deployment-specific and usually require controlled release management.
Stripe product identifiers Environment for now STRIPE_SUBSCRIPTION_PRICE_ID, webhook secrets, live/test secret keys Stripe IDs are operationally sensitive. Keep them in env until a fuller admin permission model exists.
Subscription business policy Platform UI Env fallback Trial days, plan code, plan name, price label, promotion code support, subscription required These may change commercially without needing a code deploy.
Revenue model and goals Platform UI Env fallback Revenue fee model, default dashboard currency, default monthly goal These are business rules. Practitioner-specific overrides remain in practitioner settings.
FX conversion policy Platform UI Env fallback FX provider URL, timeout, fallback rates, update schedule label, current monthly snapshot Used only for performance tracking. Actual revenue remains in original transaction currencies.
Referral programme Platform UI Env fallback Referral enabled, reward months, trigger Growth rules should be adjustable without redeployment.
Sharing and paid bundle defaults Platform UI Env fallback Default share price, default share currency, share access cookie age Commercial defaults and client-access behaviour may need quick operational adjustment.
Stripe Connect commercial policy Platform UI Env fallback Connect enabled, account type, country, fallback to platform, platform fee percentage, fixed fee Marketplace policy belongs in the control station; secret keys remain in env.
Report engine defaults Platform UI Config fallback Recommendation mode, tone, max sections, max markers, product recommendations, appendix Global defaults can be managed centrally; practitioner branding/settings can still override where applicable.
Practitioner-specific settings Practitioner settings Clinic branding, logo, cover image, support email, website URL, individual dashboard currency/monthly goal These belong to each practitioner/workspace, not the platform-level control station.
Operating rule: environment variables are for secrets, infrastructure, and emergency fallbacks; Platform Settings are for business rules; Practitioner Settings are for per-user branding and workspace preferences.
